1-1-2: エディタの選択とインストール

🎯 このセクションで学ぶこと

  • プロのフロントエンド開発で最も広く使われているコードエディタ「Visual Studio Code (VS Code)」について、その特徴と利点を理解します。
  • お使いのPC（Mac/Windows）にVS Codeをインストールし、基本的な設定を行うことができます。
  • フロントエンド開発の効率を劇的に向上させる、最低限入れておくべきVS Codeの拡張機能を知り、インストールできます。

🧠 先輩エンジニアの思考プロセス

「新しい言語を学ぶなら、まず何から始める？」と聞かれたら、多くのエンジニアは「エディタを整えること」と答えるでしょう。なぜなら、優れたエディタは単なるテキスト入力ツールではなく、コードの品質を上げ、開発スピードを加速させる「最高の相棒」だからです。

なぜこの順番で学ぶのか？

  1. 「なぜ」を理解する: まず、数あるエディタの中でなぜVS Codeが選ばれるのか、その理由を知ることで、ツールのポテンシャルを最大限に引き出す意識が生まれます。
  2. 環境を統一する: 次に、実際にインストールして、この教材全体を通して使う環境を最初に整えます。これにより、後のセクションで環境の違いによる無用なトラブルを避けることができます。
  3. 「素の状態」から「強化」へ: 最後に、拡張機能を追加することで、エディタがどのように「賢く」なっていくのかを体験します。これは、ツールを自分好みにカスタマイズしていくエンジニアの基本的なスタンスを学ぶ第一歩です。
このセクションでやること
1. なぜVisual Studio Codeなのかを知る
2. Visual Studio Codeをインストールする

💡 ポイント: 優れた道具を選ぶことは、優れた職人になるための第一歩。ツールの選定理由を理解し、自分の手に馴染ませることが重要です。

導入: あなたの「剣」を選ぼう

プログラミングの世界に足を踏み入れるあなたを、冒険に出る冒険者に例えるなら、コードエディタはあなたの「剣」です。もちろん、シンプルなメモ帳でもコードは書けますが、それはまるで「こん棒」で巨大なドラゴンに挑むようなもの。無駄な時間がかかり、困難も多いでしょう。

プロのエンジニアは、コードを色付けして見やすくしてくれたり、入力ミスを教えてくれたり、面倒な定型文を自動で入力してくれたりする、高機能な「魔法の剣」を使います。それがVisual Studio Code（通称VS Code）です。

このセクションでは、なぜ世界中の多くのフロントエンドエンジニアがこの「剣」を愛用するのかを学び、実際にあなたの手に取って、冒険の準備を整えます。

概念の説明: コードエディタとIDEの違い

プログラミングで使うツールには、大きく分けて「コードエディタ」と「IDE（統合開発環境）」があります。それぞれの特徴を簡単に見てみましょう。

項目 コードエディタ (例: VS Code, Sublime Text) IDE (例: PhpStorm, Xcode, Visual Studio)
目的 テキスト（コード）を書くことに特化 コード作成からデバッグ、ビルドまで、開発に必要な全機能を一つにまとめた環境
特徴 軽量で起動が速い。拡張機能で自分好みにカスタマイズできる。 特定の言語やフレームワークに最適化されている。多機能だが、動作が重い傾向がある。
比喩 高機能な文房具セット。必要なペンや定規を自分で選んで組み合わせる。 専用の工作機械。特定の製品を作るための全ての工具が揃っている。

Laravel開発では、多機能なIDEである「PhpStorm」を使っている方も多いかもしれません。しかし、フロントエンド開発、特にJavaScriptの世界では、軽量でカスタマイズ性の高いコードエディタであるVS Codeが圧倒的なシェアを誇ります。その理由は、フロントエンド開発で使われる技術の多様性にあります。React, Vue, Svelteといった様々なフレームワークや、CSS, TypeScript, Markdownといった各種言語を、拡張機能一つで柔軟に切り替えながら扱えるVS Codeの軽快さが、変化の速いフロントエンドの世界にマッチしているのです。

詳細解説: なぜVisual Studio Codeなのか？

VS Codeが絶大な支持を得ている理由は、主に以下の3つです。

  1. 豊富な拡張機能（Extensions）: VS Codeの最大の魅力は、世界中の開発者が作成した無数の拡張機能にあります。これにより、シンタックスハイライト（コードの色付け）、コード補完、エラーチェック、フォーマット（コードの整形）など、あらゆる言語やフレームワークに最適化された機能を追加できます。まるでスマートフォンにアプリを追加していく感覚です。

  2. IntelliSense（インテリセンス）: これは、VS Codeが持つ強力なコード補完機能です。あなたが書こうとしているコードを予測し、候補を表示してくれます。これにより、タイピングミスが減り、開発スピードが劇的に向上します。特に、後々学ぶTypeScriptとの相性は抜群です。

  3. Gitとの統合: VS Codeは、バージョン管理ツールであるGitの機能を標準で内蔵しています。エディタ上で変更したファイルを確認したり、コミット（変更の記録）したりといった基本的な操作を、画面を切り替えることなく直感的に行えます。

🏃 実践: VS Codeをインストールして設定しよう

それでは、実際にVS Codeをインストールし、フロントエンド開発のための最初の設定を行いましょう。

Step 1: Visual Studio Codeのダウンロード

1-1. まずは公式サイトにアクセスします。

Visual Studio Code 公式サイト

1-2. サイトにアクセスすると、あなたのOS（Mac/Windows）に合ったダウンロードボタンが大きく表示されています。これをクリックして、インストーラーをダウンロードしてください。

Step 2: インストール

ダウンロードしたインストーラーを実行し、画面の指示に従ってインストールを進めます。基本的に、デフォルト設定のままで問題ありません。

  • Windowsの場合: インストール中に「PATHへの追加」というチェックボックスが表示されます。これは必ずチェックしておいてください。これにより、後ほどコマンドプロンプトやPowerShellからcodeというコマンドでVS Codeを起動できるようになります。
  • Macの場合: ダウンロードしたVisual Studio Code.appを「アプリケーション」フォルダにドラッグ＆ドロップするだけでインストールは完了です。

Step 3: 日本語化

VS Codeはデフォルトでは英語表示です。英語のままでも問題ありませんが、慣れないうちは日本語で使いたい方も多いでしょう。以下の手順で簡単に日本語化できます。

3-1. VS Codeを起動します。

3-2. 左側のアクティビティバー（アイコンが縦に並んでいる部分）の一番下にある、四角が4つ組み合わさったような「拡張機能（Extensions）」アイコンをクリックします。

3-3. 上部に表示される検索ボックスに「Japanese Language Pack」と入力します。

3-4. 検索結果に「Japanese Language Pack for Visual Studio Code」が表示されるので、「Install」ボタンをクリックします。

3-5. インストールが完了すると、右下に「Restart to apply」といった内容の通知が表示されるので、ボタンをクリックしてVS Codeを再起動します。

再起動後、メニューなどが日本語で表示されていれば成功です。

Step 4: 必須の拡張機能をインストール

次に、フロントエンド開発の効率を上げる、最低限入れておきたい拡張機能をインストールしましょう。先ほどの日本語化と同様に、拡張機能の検索ボックスから以下の名前を検索し、インストールしてください。

拡張機能名 役割
Prettier - Code formatter ⚙️ コードの見た目を自動で綺麗に整えてくれるツール（フォーマッター）。チーム開発でコードのスタイルを統一するために必須です。
ESLint ⚙️ JavaScript/TypeScriptのコードを分析し、文法的な間違いや潜在的なバグを指摘してくれるツール（リンター）。コードの品質を保つための「先生」のような存在です。
Live Server ⚙️ HTML/CSSファイルを保存した瞬間に、自動でブラウザの表示を更新してくれます。手動でリロードする手間が省け、開発効率が大幅に向上します。
Material Icon Theme 🎨 ファイルの種類に応じて、エクスプローラー（ファイル一覧）のアイコンを見やすく表示してくれます。視覚的にファイルの種類を判別しやすくなります。

Step 5: 設定の変更（保存時に自動フォーマット）

最後に、ファイルを保存するたびにPrettierが自動でコードを綺麗にしてくれるように設定しましょう。この設定一つで、あなたのコードは常にプロフェッショナルな見た目に保たれます。

5-1. [Ctrl] + [,]（Macの場合は [⌘] + [,]）を押して、設定画面を開きます。

5-2. 右上のファイルのアイコン（「設定（JSON）を開く」）をクリックして、settings.jsonという設定ファイルを開きます。

5-3. settings.jsonファイルに、以下の内容を追記（または既存の内容とマージ）してください。

{
  // ...他の設定があるかもしれません

  // どの言語でもPrettierをデフォルトのフォーマッターとして使用する
  "editor.defaultFormatter": "esbenp.prettier-vscode",

  // ファイル保存時に自動でフォーマットを実行する
  "editor.formatOnSave": true
}

コードリーディング

  • "editor.defaultFormatter": "esbenp.prettier-vscode"
    • → VS Codeに内蔵されているエディタ機能に対して、「デフォルトのフォーマッター（コード整形ツール）として、Prettier（esbenp.prettier-vscode）を使ってください」と命令しています。
  • "editor.formatOnSave": true
    • → 「ファイルを保存（Save）した時に（On）、自動でフォーマット（Format）を実行してください」という設定です。trueは「はい、実行します」という意味です。

これで、あなたがファイルを保存するたびに、VS CodeがPrettierを使ってコードを自動的に整形してくれます。

✨ まとめ

  • VS Codeの強み: フロントエンド開発では、軽量でカスタマイズ性の高いVS Codeが主流です。その強みは「豊富な拡張機能」「強力なコード補完（IntelliSense）」「Gitとの統合」にあります。
  • インストールの手順: 公式サイトからダウンロードし、インストーラーの指示に従うだけで簡単にインストールできます。
  • 日本語化と拡張機能: 「Japanese Language Pack」で日本語化し、「Prettier」「ESLint」「Live Server」といった必須の拡張機能を導入することで、開発環境を強化しました。
  • 自動フォーマット設定: settings.jsonを編集し、ファイル保存時にコードが自動で整形されるように設定しました。これにより、常に綺麗なコードを保つことができます。
