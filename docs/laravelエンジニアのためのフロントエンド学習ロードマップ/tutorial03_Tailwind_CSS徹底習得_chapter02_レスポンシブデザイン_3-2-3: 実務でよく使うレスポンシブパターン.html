3-2-3: 実務でよく使うレスポンシブパターン

Chapter 2: レスポンシブデザイン

Section 3: 実務でよく使うレスポンシブパターン

🎯 このセクションで学ぶこと

  • レスポンシブなナビゲーションメニュー（ハンバーガーメニュー）の基本的な考え方を理解する。
  • レスポンシブなグリッドレイアウトを組めるようになる。
  • 特定のブレークポイントで要素を表示・非表示させる方法を習得する。

イントロダクション：レスポンシブデザインの「型」を学ぶ

モバイルファーストの考え方とブレークポイントの使い方を学んだ今、実務で頻繁に登場する具体的なレスポンシブデザインのパターンをいくつか見ていきましょう。

これらの「型」を覚えておくことで、ゼロからレイアウトを考える時間を大幅に短縮し、より効率的にUIを構築できるようになります。このセクションでは、特に重要な3つのパターン「レスポンシブ・ナビゲーション」「レスポンシブ・グリッド」「要素の表示/非表示」に焦点を当てます。

⚙️ パターン1: レスポンシブ・ナビゲーション（ハンバーガーメニュー）

PCではヘッダーにメニュー項目が横並びになっていますが、スマートフォンではスペースが足りないため、ハンバーガーアイコン（≡）をタップするとメニューが開く、というUIは非常によく見られます。

Tailwind CSSだけでこの開閉機能を実装するには少しJavaScriptが必要になりますが、ここではそのレイアウトの切り替え部分に焦点を当てます。

<nav class="bg-gray-800 p-4">
  <div class="container mx-auto flex justify-between items-center">
    <!-- ロゴ -->
    <div class="text-white font-bold text-xl">MySite</div>

    <!-- PC用メニュー (md以上で表示) -->
    <div class="hidden md:flex space-x-4">
      <a href="#" class="text-gray-300 hover:text-white">Home</a>
      <a href="#" class="text-gray-300 hover:text-white">About</a>
      <a href="#" class="text-gray-300 hover:text-white">Contact</a>
    </div>

    <!-- ハンバーガーアイコン (md未満で表示) -->
    <div class="md:hidden">
      <button class="text-white">
        <svg class="w-6 h-6" fill="none" stroke="currentColor" viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M4 6h16M4 12h16m-7 6h7"></path></svg>
      </button>
    </div>
  </div>
</nav>
  • コードリーディング
    • PC用メニュー:
      • hidden: これがモバイルの基本スタイルです。デフォルトではメニューを非表示にしています。
      • md:flex: 画面幅がmd（768px）以上になると、display: flex;が適用され、メニューが表示されるようになります。（hiddenはdisplay: none;なので、displayプロパティが上書きされる形）
    • ハンバーガーアイコン:
      • md:hidden: 画面幅がmd（768px）以上になると、display: none;が適用され、ハンバーガーアイコンが非表示になります。
      • プレフィックスなしのクラスがないため、デフォルト（モバイル）では表示されます。

このように、hiddenとmd:flex（またはmd:blockなど）を組み合わせることで、特定のブレークポイントを境に要素の表示/非表示を切り替えることができます。

⚙️ パターン2: レスポンシブ・グリッド

商品一覧ページやブログ記事一覧などでよく使われる、画面サイズに応じてカラム数を変更するグリッドレイアウトです。これは前のセクションでも少し触れましたが、非常に重要なパターンです。

<div class="container mx-auto p-4">
  <div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4">
    <!-- カード1 -->
    <div class="bg-white p-4 rounded-lg shadow">カードコンテンツ</div>
    <!-- カード2 -->
    <div class="bg-white p-4 rounded-lg shadow">カードコンテンツ</div>
    <!-- カード3 -->
    <div class="bg-white p-4 rounded-lg shadow">カードコンテンツ</div>
    <!-- カード4 -->
    <div class="bg-white p-4 rounded-lg shadow">カードコンテンツ</div>
    <!-- ...さらにカードが続く -->
  </div>
</div>
  • コードリーディング
    • grid-cols-1: モバイル（デフォルト）では1列。
    • sm:grid-cols-2: sm（640px）以上で2列。
    • lg:grid-cols-3: lg（1024px）以上で3列。
    • xl:grid-cols-4: xl（1280px）以上で4列。

モバイルファーストの考え方に基づき、小さい画面から大きい画面へと、カラム数を追加していくように定義しているのがポイントです。

⚙️ パターン3: レスポンシブな余白とフォントサイズ

画面が大きくなるにつれて、セクション間の余白（マージン）や、見出しのフォントサイズを大きくして、全体のバランスを取りたい場合があります。

<div class="py-8 md:py-12 lg:py-16">
  <h1 class="text-3xl md:text-4xl lg:text-5xl font-bold">
    レスポンシブな見出し
  </h1>
  <p class="mt-4 text-base md:text-lg">
    画面サイズに応じて、このセクションの上下の余白と、見出しのフォントサイズが変わります。
  </p>
</div>
  • コードリーディング
    • py-8 md:py-12 lg:py-16: 親divの垂直方向（上下）のパディングを、モバイルではpy-8、md以上でpy-12、lg以上でpy-16と、段階的に大きくしています。
    • text-3xl md:text-4xl lg:text-5xl: h1タグのフォントサイズも同様に、画面が大きくなるにつれて段階的に大きくしています。

これにより、どの画面サイズで見ても間延びしたり、窮屈になったりすることのない、バランスの取れたデザインを維持することができます。

✨ まとめ

  • hiddenとブレークポイントプレフィックス付きのdisplayクラス（例: md:flex）を組み合わせることで、要素の表示/非表示をレスポンシブに切り替えられる。
  • grid-cols-*とブレークポイントプレフィックスを組み合わせることで、レスポンシブなグリッドレイアウトを簡単に実装できる。
  • 余白（p*, m*）やフォントサイズ（text-*）も、ブレークポイントプレフィックスを使ってレスポンシブに対応させることが、デザインのバランスを保つ上で重要である。
