Chapter 1: HTMLの基礎

Chapter 1 ハンズオン: 簡単な自己紹介ページを作成する

🖼️ 完成形のイメージ

このハンズオンでは、このチャプターで学んだHTMLタグを総動員して、以下のようなシンプルな自己紹介ページをゼロから作成します。この経験を通じて、HTMLがWebページの「骨格」をどのように作っているのかを体感的に理解します。

ここに完成した自己紹介ページのスクリーンショットを挿入: 見出し、画像、自己紹介文、SNSリンクのリストが構造的に配置されている様子

🎯 このセクションで学ぶこと

このハンズオンを終えると、あなたは以下のことができるようになります。

  • このチャプターで学んだ主要なHTMLタグ（<h1>, <p>, <img>, <a>, <ul>, <li>, <div>）を組み合わせて、簡単なWebページをゼロから作成できるようになる。
  • HTMLファイルの基本的な構造を、何も見ずに記述できるようになる。
  • <div> を使って関連するコンテンツをグループ化する、という実践的な使い方を体験する。

🏃 実践: 一緒に作ってみましょう！

理論を学んだら、次は手を動かして形にする番です。このチャプターで学んだ知識を総動員して、あなたの分身となる簡単な「自己紹介ページ」をHTMLだけで作成してみましょう。CSSを使わないため、見た目は非常にシンプルですが、Webページの「骨格」をしっかりと組み立てる練習になります。

💭 実装の思考プロセス

Webページ作成の第一歩は、まず「どんな情報を載せたいか」を考え、それをHTMLの構造に落とし込むことです。いきなりコードを書き始めるのではなく、頭の中で簡単な設計図を描きます。

このハンズオンのゴール:

  1. 情報の整理: 自己紹介に必要な情報（名前、自己紹介文、プロフィール画像、SNSリンクなど）を洗い出す。
  2. 構造化: 洗い出した情報を、見出し、段落、画像、リストといった適切なHTMLタグに割り当てる。
  3. グループ化: 関連性の高い情報（例えば、プロフィール全体）を <div> で一つのかたまりとしてまとめる。
  4. コーディング: 設計図に基づいて、実際にHTMLファイルを記述していく。

💡 ポイント: HTMLは「情報の設計図」。どの情報が一番重要で（<h1>）、どの情報が補足で（<p>）、どの情報がリストなのか（<ul>）を意味づけしていく作業がHTMLコーディングの本質です。

📝 ステップバイステップで実装

Step 1: HTMLファイルの基本構造を作成する

まずは、すべてのHTMLの土台となる基本構造を記述したファイルを作成します。

  • 何を考えているか: 「何はともあれ、まずはHTMLのお作法である <!DOCTYPE> から始まる基本のテンプレートを書かないと始まらない。ファイル名は index.html にしておこう。これがWebサイトの玄関になるファイル名の慣習だからだ。」

1-1. VSCodeなどのエディタで、新しいファイル index.html を作成します。

1-2. 以下の基本構造を記述します。lang属性を ja に、<title> を「自己紹介」に変更しておきましょう。

  • コマンド/コード:
    html <!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <title>自己紹介</title> </head> <body> <!-- ここにコンテンツを追加していく --> </body> </html>
  • コマンド解説/コードリーディング:
    • <!DOCTYPE html> → このファイルがHTML5であることを宣言しています。
    • <html lang="ja"> → ページの言語が日本語であることを示します。
    • <meta charset="UTF-8"> → 文字コードをUTF-8に指定し、文字化けを防ぎます。
    • <title>自己紹介</title> → ブラウザのタブに「自己紹介」と表示されます。
    • <body> → これから記述する、ページに表示されるすべての内容がこの中に入ります。
Step 2: プロフィール全体を囲むdivと見出しを追加する

次に、自己紹介コンテンツ全体を一つの大きな箱としてまとめるために <div> を配置し、その中に最も重要な見出しとして自分の名前を <h1> で記述します。

  • 何を考えているか: 「自己紹介の情報を全部まとめて一つの『カード』みたいにしたいな。後でCSSでスタイルを当てることを見越して、全体を <div> で囲っておこう。そして、このページの主役である自分の名前を <h1> で一番目立たせるぞ。」

2-1. <body> タグの中に、<div> タグを追加します。

2-2. <div> の中に、<h1> タグを使って自分の名前を記述します。

  • コマンド/コード:
    html <body> <div> <h1>山田 太郎</h1> </div> </body>
Step 3: プロフィール画像と自己紹介文を追加する

見出しの下に、プロフィール画像と簡単な自己紹介文を追加します。画像は <img> タグ、自己紹介文は <p> タグを使います。

  • 何を考えているか: 「名前の次には顔写真と自己紹介文を置くのが自然だろう。画像は <img> タグ、文章は <p> タグ。これが基本だ。画像ファイルは images フォルダを作ってそこに入れておこう。alt属性も忘れずに書かないと。」

3-1. index.html と同じ階層に images というフォルダを作成し、その中に表示したい画像ファイル（例: profile.png）を保存します。

3-2. <h1> タグの下に、<img> タグと <p> タグを追加します。

  • コマンド/コード:
    html <body> <div> <h1>山田 太郎</h1> <img src="images/profile.png" alt="山田太郎のプロフィール画像" width="150"> <p>こんにちは！LaravelとVue.jsを使った開発が得意なWebエンジニアです。最近はフロントエンドのモダンな技術、特にReactとNext.jsを学習しています。</p> <p>趣味は週末のキャンプとコーヒーを淹れることです。</p> </div> </body>
  • コマンド解説/コードリーディング:
    • <img src="images/profile.png" ...> → images フォルダの中にある profile.png を表示します。
    • alt="..." → 画像が表示できない場合に、このテキストが表示されます。
    • width="150" → 画像の表示幅を150ピクセルに指定しています。（CSSで指定するのが一般的ですが、HTMLでも指定可能です）
Step 4: SNSリンクのリストを追加する

最後に、自分のSNSアカウントなどへのリンクをリスト形式で追加します。見出しとして <h2> を使い、リンクのリストは <ul>, <li>, <a> を組み合わせて作成します。

  • 何を考えているか: 「SNSリンクはいくつかあるから、リスト形式が見やすいな。順序は関係ないから <ul> を使おう。各項目は <li> で、その中に <a> タグでリンクを設置する。セクションの見出しとして <h2> を使うのが構造的にも正しいだろう。」

4-1. 自己紹介文の <p> タグの下に、<h2> タグと <ul> タグを追加します。

  • コマンド/コード:
    ```html


    山田 太郎



    こんにちは！...


    趣味は...

    <h2>SNS & Links</h2>
<ul>
  <li><a href="https://github.com/your-account" target="_blank">GitHub</a></li>
  <li><a href="https://twitter.com/your-account" target="_blank">Twitter</a></li>
  <li><a href="https://your-blog.com" target="_blank">ブログ</a></li>
</ul>



    `` - **コマンド解説/コードリーディング:** -

    SNS & Links

    → このセクションが見出しレベル2であることを示します。 -
      → 順序のないリストを開始します。 -
    • ...
      • → 各リスト項目の中に、リンクを配置しています。 -target="_blank"` → リンクをクリックしたときに、新しいタブでページが開くようにする属性です。

    ✨ 完成！

    お疲れ様でした！index.html をブラウザで開いてみてください。CSSが適用されていないため非常に質素ですが、見出し、画像、文章、リンクのリストが構造的に配置された、立派なWebページが完成しました。あなたは今、HTMLだけでWebページの骨格をゼロから組み立てるスキルを身につけたのです。

    💡 ヒント

    • Live Server (VSCode拡張機能): VSCodeの拡張機能「Live Server」をインストールすると、HTMLファイルを保存するたびにブラウザが自動でリロードされるようになり、開発効率が劇的に向上します。
    • ダミー画像: ちょうど良い画像がない場合は、「placeholder image」などで検索すると、ダミー画像を生成してくれるサービスが見つかります。例えば https://via.placeholder.com/150 のようなURLを <img> の src に指定するだけで、150x150のダミー画像を表示できます。

    📖 模範解答

    完成した index.html の全体像は以下のようになります。

    <!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>自己紹介</title>
</head>
<body>
  <div>
    <h1>山田 太郎</h1>
    <img src="https://via.placeholder.com/150" alt="山田太郎のプロフィール画像" width="150">
    <p>こんにちは！LaravelとVue.jsを使った開発が得意なWebエンジニアです。最近はフロントエンドのモダンな技術、特にReactとNext.jsを学習しています。</p>
    <p>趣味は週末のキャンプとコーヒーを淹れることです。</p>

    <h2>SNS & Links</h2>
    <ul>
      <li><a href="#" target="_blank">GitHub</a></li>
      <li><a href="#" target="_blank">Twitter</a></li>
      <li><a href="#" target="_blank">ブログ</a></li>
    </ul>
  </div>
</body>
</html>

    （imgのsrcとaのhrefはダミーのものに置き換えています）
