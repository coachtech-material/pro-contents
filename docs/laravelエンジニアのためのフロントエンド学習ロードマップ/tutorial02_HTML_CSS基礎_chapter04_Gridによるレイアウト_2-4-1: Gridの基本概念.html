Chapter 4: Gridによるレイアウト

Section 1: Gridの基本概念

🖼️ 完成形のイメージ

このセクションでは、Flexboxと並ぶもう一つの強力なレイアウトシステム、「CSS Grid Layout」の基本を学びます。Gridは、Flexboxが一次元（縦か横か）のレイアウトを得意とするのに対し、行と列からなる二次元（縦と横を同時）のレイアウトを組むことを得意とします。

ここにスクリーンショットを挿入: 9つのボックスが3x3の格子状にきれいに並んでいる様子

🎯 このセクションで学ぶこと

このセクションでは、CSS Gridの基本的な概念と、Flexboxとの違いを理解します。このセクションを終えると、あなたは以下のことができるようになります。

  • Gridが二次元レイアウトシステムであることを説明できるようになる。
  • Gridレイアウトの基本構造である「親要素（Gridコンテナ）」と「子要素（Gridアイテム）」の関係を理解する。
  • display: grid; を使って、Gridレイアウトを有効にできるようになる。
  • FlexboxとGridの使い分けについて、自分なりの考えを述べられるようになる。

導入

Flexboxは要素を一列に並べるのが得意でしたが、Webページのレイアウトはもっと複雑です。ヘッダー、サイドバー、メインコンテンツ、フッターといった、ページ全体を大きな格子状に区切るようなレイアウトを考えた時、Flexboxだけでは複雑な入れ子構造が必要になり、コードが煩雑になりがちです。CSS Gridは、このような二次元的なレイアウトを、より直感的かつ強力に制御するために設計されました。Excelのシートのように、ページを格子状のエリアに分割し、そこにコンテンツを配置していくイメージです。

概念の説明

Gridの基本構造も、Flexboxと同様に「親要素」と「子要素」の関係で成り立っています。

  1. 親要素 (Grid Container / グリッドコンテナ):

    • レイアウトの土台となる格子を定義する親要素です。
    • この親要素に display: grid; を指定することで、Gridレイアウトが有効になります。

  2. 子要素 (Grid Item / グリッドアイテム):

    • Gridコンテナの直下にある子要素のことです。
    • これらのアイテムが、コンテナによって作られたグリッドの各セルに配置されます。

図解：Gridの親子関係

<div class="grid-container"> <!-- 親要素 -->
  <div class="grid-item">アイテム1</div> <!-- 子要素 -->
  <div class="grid-item">アイテム2</div> <!-- 子要素 -->
  <div class="grid-item">アイテム3</div> <!-- 子要素 -->
  ...
</div>

.grid-container {
  display: grid; /* これでGridレイアウトが有効になる */
  /* ここに「どんな格子を作るか」を定義していく */
}

display: grid; を指定しただけでは、見た目はブロック要素が縦に並んでいるだけと変わりません。Gridレイアウトの本質は、ここから「どのような行と列（トラック）を作るか」を定義していく点にあります。この行と列によって作られる一つ一つのマス目を「セル」と呼びます。

FlexboxとGridの使い分け

どちらも強力なレイアウトツールですが、得意なことが異なります。

  • Flexbox → 一次元レイアウト

    • アイテムを一列（または一行）に並べ、その中での揃えや間隔を調整するのが得意。
    • コンポーネント内のパーツ（例: ボタンと入力欄、アイコンとテキスト）のレイアウトに向いている。
    • コンテンツ主導のレイアウト。アイテムのサイズに応じて全体のレイアウトが決まる。

  • Grid → 二次元レイアウト

    • ページ全体を行と列で格子状に分割し、そのエリアに要素を配置していくのが得意。
    • ページ全体の大きな骨格（例: ヘッダー、サイドバー、メイン、フッター）のレイアウトに向いている。
    • レイアウト主導。先に格子状のレイアウトを定義し、そこにコンテンツを流し込む。

実務では、ページ全体の骨格をGridで作り、その中の各コンポーネントのレイアウトをFlexboxで作る、というように両者を組み合わせて使うのが一般的です。

💡 TIP

  • Gridデバッグツール: ChromeやFirefoxの開発者ツールには、Gridレイアウトを視覚的にデバッグするための非常に優れた機能があります。Gridコンテナにマウスを合わせると、定義したグリッド線やエリアがハイライト表示され、レイアウトの構造を直感的に理解できます。Gridを学ぶ上では必須のツールです。

✨ まとめ

  • CSS Gridは、行と列からなる二次元のレイアウトを構築するためのシステムである。
  • 親要素に display: grid; を指定して「Gridコンテナ」を作る。
  • Gridコンテナの直下の子要素は「Gridアイテム」となり、定義されたグリッドのセルに配置される。
  • Flexboxは一次元（線）、Gridは二次元（面）のレイアウトを得意とし、実務では両者を組み合わせて使う。
