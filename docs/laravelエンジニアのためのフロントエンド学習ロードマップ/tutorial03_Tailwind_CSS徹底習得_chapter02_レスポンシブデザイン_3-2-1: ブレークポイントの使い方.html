3-2-1: ブレークポイントの使い方

Chapter 2: レスポンシブデザイン

Section 1: ブレークポイントの使い方

🎯 このセクションで学ぶこと

  • レスポンシブデザインがなぜ現代のWeb開発で必須なのかを説明できるようになる。
  • Tailwind CSSのデフォルトブレークポイント（sm, md, lg, xl, 2xl）を理解する。
  • ブレークポイントプレフィックス（例: md:）を使って、特定の画面サイズ以上でスタイルを適用できるようになる。

イントロダクション：あらゆる画面サイズに対応するということ

現代では、WebサイトはデスクトップPCだけでなく、スマートフォン、タブレット、大型モニターなど、多種多様なデバイスで閲覧されます。レスポンシブデザインとは、閲覧しているデバイスの画面サイズに応じて、Webページのレイアウトやデザインを最適化する手法のことです。

これができないと、スマートフォンでPCサイトを表示したときに文字が小さすぎて読めなかったり、逆にPCでスマホサイトを表示したときにコンテンツが中央に寄ってスカスカに見えたり、といった問題が起こります。

Tailwind CSSは、このレスポンシブデザインを驚くほど簡単かつ直感的に実装できる仕組みを提供します。それがブレークポイントプレフィックスです。

🔑 ブレークポイントとは？

ブレークポイントとは、デザインが切り替わる画面幅の「境界線」のことです。例えば、「画面幅が768px以上になったら、サイドバーを表示する」といったルールを定義する際の「768px」がブレークポイントにあたります。

Tailwind CSSには、一般的なデバイスの画面サイズに基づいた5つのデフォルトブレークポイントが用意されています。

プレフィックス 最小幅 CSSメディアクエリ 対象デバイス（目安）
sm 640px @media (min-width: 640px) スマートフォン（横向き）
md 768px @media (min-width: 768px) タブレット
lg 1024px @media (min-width: 1024px) ノートPC
xl 1280px @media (min-width: 1280px) デスクトップPC
2xl 1536px @media (min-width: 1536px) 大型モニター

重要なポイント: Tailwindのブレークポイントはすべて「最小幅（min-width）」で定義されています。これは、md:bg-blue-500というクラスが「画面幅が768px以上の場合に背景を青にする」という意味になることを示します。この「以上」という考え方が、次のセクションで学ぶ「モバイルファースト」の思想と密接に結びついています。

⚙️ ブレークポイントプレフィックスの使い方

使い方は非常にシンプルです。適用したいユーティリティクラスの前に、{breakpoint}:というプレフィックスを付けるだけです。

例：画面サイズに応じて背景色を変える

<div class="bg-red-500 md:bg-blue-500 lg:bg-green-500">
  <!-- ... -->
</div>

このdiv要素の背景色は、画面幅に応じて以下のように変化します。

  • 画面幅が 768px 未満の場合: bg-red-500が適用され、背景は赤色になります。（プレフィックスなしのクラスは、すべての画面サイズで適用されます）
  • 画面幅が 768px 以上 1024px 未満の場合: md:bg-blue-500がbg-red-500を上書きし、背景は青色になります。
  • 画面幅が 1024px 以上の場合: lg:bg-green-500がmd:bg-blue-500を上書きし、背景は緑色になります。

例：画面サイズに応じてレイアウトを変更する

レスポンシブデザインで最もよく使われるのが、レイアウトの変更です。例えば、スマートフォンでは要素を縦に並べ、タブレット以上では横に並べたい場合、以下のように記述できます。

<div class="md:flex">
  <div class="md:w-1/2">要素A</div>
  <div class="md:w-1/2">要素B</div>
</div>
  • 画面幅が 768px 未満の場合:
    • md:flexやmd:w-1/2は適用されません。
    • divはデフォルトのブロック要素として振る舞うため、要素Aと要素Bは縦に並びます。
  • 画面幅が 768px 以上の場合:
    • md:flexが適用され、親要素がFlexコンテナになります。
    • md:w-1/2が適用され、子要素の幅がそれぞれ50%になります。
    • 結果として、要素Aと要素Bは横に並びます。

このように、HTMLのクラスを少し変更するだけで、CSSファイルにメディアクエリを一行も書くことなく、複雑なレスポンシブ対応が実現できてしまうのが、Tailwind CSSの最大の強みの一つです。

✨ まとめ

  • レスポンシブデザインとは、画面サイズに応じてデザインを最適化する手法である。
  • Tailwind CSSでは、sm, md, lg, xl, 2xlというブレークポイントが定義されている。
  • ユーティリティクラスの前にmd:のようなプレフィックスを付けることで、特定の画面幅以上でスタイルを適用できる。
  • Tailwindのブレークポイントはすべて「最小幅（min-width）」で定義されており、「モバイルファースト」のアプローチと相性が良い。
