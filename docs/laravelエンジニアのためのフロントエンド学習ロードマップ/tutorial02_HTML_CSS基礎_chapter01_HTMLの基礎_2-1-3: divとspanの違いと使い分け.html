2-1-3: divとspanの違いと使い分け

Chapter 1: HTMLの基礎

Section 3: divとspanの違いと使い分け

🖼️ 完成形のイメージ

このセクションでは、<div>と<span>の役割を学び、以下のようにページのセクションをグループ化したり、文中の特定の部分だけスタイルを変えたりする方法を理解します。

ここにスクリーンショットを挿入: divで囲まれたセクションが背景色で区切られ、spanで囲まれたテキストの色が変わっている様子がわかる

🎯 このセクションで学ぶこと

このセクションでは、CSSやJavaScriptで要素を扱うための「グループ化」に使う、<div>と<span>タグについて学びます。このセクションを終えると、あなたは以下のことができるようになります。

  • ブロックレベル要素とインライン要素の振る舞いの違いを説明できるようになる。
  • <div>と<span>を、それぞれの特性に合わせて適切に使い分けられるようになる。
  • CSSでスタイルを適用するために、HTML要素をグループ化する必要性を理解する。

導入

HTMLには、<h1> や <p> のようにそれ自体が明確な意味を持つタグの他に、特に意味を持たない汎用的なタグが存在します。それが <div> と <span> です。これらは、CSSでスタイルを適用するためや、JavaScriptで特定の範囲を操作するために、要素をグループ化する目的で使われます。一見地味ですが、実務では最も頻繁に使うタグと言っても過言ではありません。この二つの違いを正確に理解することは、レイアウトを組む上で非常に重要です。

概念の説明

<div> と <span> の最大の違いは、ブロックレベル要素であるか、インライン要素であるか、という点にあります。

ブロックレベル要素 (Block-level Element)

  • 特徴: 一つの「塊（ブロック）」として扱われ、前後に改行が入ります。横幅は、特に指定しなければ親要素の幅いっぱいに広がります。高さは、中のコンテンツの量によって決まります。
  • 代表例: <div>, <h1>〜<h6>, <p>, <ul>, <ol>, <li> など。
  • イメージ: 書類棚の「引き出し」のようなものです。一つの引き出しが一行を占有し、その中に書類（コンテンツ）を入れます。

インライン要素 (Inline Element)

  • 特徴: 文章の一部として扱われ、前後に改行が入りません。横幅と高さは、中のコンテンツの量によって決まります。幅や高さをCSSで指定しても、基本的には無視されます。
  • 代表例: <span>, <a>, <img>, <strong> (太字), <em> (斜体) など。
  • イメージ: 文章中の「マーカー」のようなものです。文の流れを断ち切ることなく、特定の部分だけをマークします。

詳細解説

この違いを理解するために、具体的なコードを見てみましょう。

<div> の振る舞い

<div> はブロックレベル要素です。複数の要素をまとめて一つのグループとして扱いたい場合によく使われます。例えば、ユーザーのプロフィール情報をまとめるカードのようなUIを作る場合などです。

  • コード例:
    html <div style="background-color: lightgray;"> <h2>ユーザー名</h2> <p>自己紹介文です。</p> </div> <div style="background-color: lightblue;"> <p>これは次のブロックです。</p> </div>
  • 表示結果:
    • 背景色がライトグレーの <div> と、ライトブルーの <div> は、それぞれが独立した行に表示されます（前後に改行が入る）。
    • 各 <div> は、横幅いっぱいに広がります。

<span> の振る舞い

<span> はインライン要素です。文章中の一部分だけ色を変えたり、太字にしたりと、特定のテキストにだけスタイルを適用したい場合に使われます。

  • コード例:
    html <p> この文章の <span style="color: red; font-weight: bold;">この部分だけ</span> スタイルを変えたい。 </p>
  • 表示結果:
    • <span> で囲まれた「この部分だけ」というテキストは、前後の文章と同じ行の中に表示されます（改行が入らない）。
    • 文字が赤くなり、太字になりますが、文全体の流れは維持されます。

使い分けのまとめ

タグ 要素の種類 特徴 主な用途
<div> ブロックレベル ・前後に改行が入る
・横幅は親要素いっぱい		 ・複数の要素をまとめるグループ化
・ページの大きなレイアウト構成
<span> インライン ・前後に改行が入らない
・幅と高さはコンテンツ次第		 ・文章中の一部分にスタイルを適用

💡 TIP

  • 入れ子のルール: ブロックレベル要素の中に、他のブロックレベル要素やインライン要素を入れることはできます。しかし、原則としてインライン要素の中にブロックレベル要素を入れることはできません。（例: <span> の中に <div> を入れるのはNG）
  • CSSでの変更: CSSの display プロパティを使えば、ブロックレベル要素をインラインのように振る舞わせたり、その逆にしたりすることも可能です。これは後のCSSのチュートリアルで詳しく学びます。

✨ まとめ

  • <div> と <span> は、それ自体に意味を持たない汎用的なタグである。
  • <div> はブロックレベル要素で、改行を伴う「塊」を作る。
  • <span> はインライン要素で、文章の流れを壊さない「部分」を作る。
  • 大きなレイアウトやグループ化には <div> を、文中の特定の部分には <span> を使うのが基本的な使い分けである。
