Chapter 4: Gridによるレイアウト

Chapter 4 ハンズオン: Gridでプロフィールカードの内部レイアウトを再構築する

🖼️ 完成形のイメージ

このハンズオンでは、これまでFlexboxで部分的に整えてきたプロフィールカードの内部レイアウトを、CSS Gridを使ってより堅牢で意味的に分かりやすい構造に再構築します。grid-template-areasを使い、カード内を「画像エリア」「自己紹介エリア」「SNSリンクエリア」に分割し、各要素を配置します。

ここに完成したプロフィールページのスクリーンショットを挿入: 見た目は大きく変わらないかもしれないが、開発者ツールでGridのオーバーレイを表示すると、カード内が綺麗なグリッドで分割されている様子がわかる

🎯 このセクションで学ぶこと

このハンズオンを終えると、あなたは以下のことができるようになります。

  • 既存のコンポーネントのレイアウトを、FlexboxからGridにリファクタリングできるようになる。
  • grid-template-columns, grid-template-rows, grid-template-areasを組み合わせて、コンポーネントレベルの二次元レイアウトを定義できるようになる。
  • grid-areaを使って、各要素を意図したグリッドセルに配置できるようになる。

🏃 実践: 一緒に作ってみましょう！

現在のプロフィールカードは、Flexboxやmargin: autoなどを駆使してレイアウトされていますが、これは部分的な最適化の集まりとも言えます。ここでは、カード全体のレイアウトをGridという一つの大きな設計図に基づいて再定義し、より管理しやすく、意味的に明確な構造へと進化させます。

💭 実装の思考プロセス

このハンズオンのゴール:

  1. レイアウトの再定義: プロフィールカード(id="profile-card")をGridコンテナとし、内部を「画像」「名前」「自己紹介文」「SNSセクション」の4つのエリアに分割する設計図をgrid-template-areasで描く。
  2. アイテムの配置: 各要素（<img>, <h1>, <p>, <h2>と<ul>をまとめたdiv）にgrid-areaを指定し、設計図通りの位置に配置する。
  3. 既存スタイルの調整: Grid化に伴い不要になったり、調整が必要になったりした既存のCSS（marginなど）を整理する。

💡 ポイント: 見た目が同じでも、その裏側にある構造が美しいかどうかは、将来のメンテナンス性に大きく影響します。Gridを使ってレイアウトの意図を明確にコードで表現する、という感覚を掴みましょう。

📝 ステップバイステップで実装

Step 1: HTMLの構造をGrid用に少し変更する

grid-areaで配置するために、SNSセクション（<h2>と<ul>）を一つの<div>でグループ化しておくと便利です。

  • 何を考えているか: 「grid-areaは一つの要素に一つのエリアを割り当てるのが基本。SNSのh2とulはセットで『SNSエリア』に配置したいから、これらをdivで囲んで一つのGridアイテムにしてしまおう。」

1-1. index.htmlのSNSセクションを、class="social-section"を持つ<div>で囲みます。

  • コマンド/コード (index.html):
    ```html

    SNS & Links


      ...



    SNS & Links


      ...


    ```

Step 2: プロフィールカードをGridコンテナにする

#profile-cardにGrid関連のプロパティを追加し、レイアウトの設計図を定義します。

  • 何を考えているか: 「カード内を、上から『名前』、その下に『画像』と『自己紹介文』を並べ、最下部に『SNSセクション』を配置する2列3行のグリッドにしよう。grid-template-areasでこの構造を視覚的に表現するぞ。」

2-1. style.cssの#profile-cardセレクタに、display: grid;と関連プロパティを追加します。また、Gridで中央揃えを行うため、既存のtext-alignやmarginを調整します。

  • コマンド/コード (style.css):
    css #profile-card { /* ...既存のスタイル... */ display: grid; /* Gridコンテナにする */ grid-template-columns: 150px 1fr; /* 左列150px, 右列は残り全部 */ grid-template-rows: auto 1fr auto; /* 3行。中段は可変高 */ grid-template-areas: "name name" "image intro" "social social"; gap: 20px; /* エリア間の隙間 */ /* Grid化に伴い不要になる、または変更するスタイル */ padding: 30px; /* paddingは維持 */ }
Step 3: 各要素をGrid Areaに配置する

各要素にgrid-areaを指定して、設計図の所定の位置に送り込みます。

  • 何を考えているか: 「HTMLの各要素に、さっき設計図で定義したエリア名をgrid-areaで割り当てていく。h1はnameエリアへ、imgはimageエリアへ...という感じだ。これに伴って、これまで中央揃えのために使っていたmargin: autoなどは不要になるから削除しよう。」

3-1. style.cssに以下のコードを追加し、既存のスタイルを修正します。

  • コマンド/コード (style.css):
    ```css
    / 既存のh1スタイルを修正 /
    h1 {
    grid-area: name; / nameエリアに配置 /
    text-align: center;
    color: #2c3e50;
    margin: 0; / Gridで配置するので不要なmarginはリセット /
    }

    / 既存の.profile-imageスタイルを修正 /
    .profile-image {
    grid-area: image; / imageエリアに配置 /
    width: 150px; / 幅を列のサイズに合わせる /
    height: 150px; / 高さも指定して正円を維持 /
    border-radius: 50%;
    / display: block; margin: auto; は不要になるので削除 /
    }

    / 自己紹介文を配置するための新しいセレクタ /

    profile-card > p {

    grid-area: intro; / introエリアに配置 /
    margin: 0; / 不要なmarginはリセット /
    }

    / SNSセクションを配置するための新しいセレクタ /
    .social-section {
    grid-area: social; / socialエリアに配置 /
    }

    / 既存のh2スタイルを修正 /
    h2 {
    margin-top: 0; / Gridのgapで余白ができるのでリセット /
    / ...その他のスタイルは維持... /
    }
    ```

✨ 完成！

お疲れ様でした！ブラウザをリロードしてください。見た目はほとんど変わらないかもしれませんが、開発者ツールで#profile-cardを選択し、Gridのオーバーレイを表示してみてください。カード内が綺麗に定義したエリアに分割され、各要素がその中に収まっているのが確認できるはずです。

あなたは今、コンポーネントのレイアウトをより構造的かつ宣言的に定義する強力なスキルを身につけました。これにより、複雑なレイアウトでも、その意図が誰にでも明確に伝わる、メンテナンス性の高いコードを書くことができるようになります。
