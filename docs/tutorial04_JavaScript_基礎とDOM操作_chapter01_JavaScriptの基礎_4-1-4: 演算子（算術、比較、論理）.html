4-1-3: 演算子（算術、比較、論理）

Chapter 1: JavaScriptの基礎

Section 3: 演算子（算術、比較、論理）

🎯 このセクションで学ぶこと

  • 算術演算子を使って、数値の四則演算や剰余計算ができるようになる。
  • 比較演算子を使って、2つの値の関係（等しい、大きいなど）を評価し、真偽値（true/false）を得られるようになる。
  • ==と===の重要な違いを説明できるようになる。
  • 論理演算子を使って、複数の条件を組み合わせることができるようになる。

イントロダクション：計算と判断の道具たち

プログラミングとは、突き詰めれば「データの加工」と「状況に応じた判断」の連続です。その最も基本的な道具となるのが演算子です。

演算子とは、+や-、>や==といった記号のことで、計算を行ったり、値を比較したり、条件を組み合わせたりする役割を持ちます。PHPにももちろん同じ概念があり、多くの演算子はJavaScriptと共通しています。しかし、特に比較演算子においては、JavaScript特有の重要な注意点も存在します。

このセクションでは、計算、比較、論理という3つのカテゴリに分けて、JavaScriptの主要な演算子を学んでいきましょう。

⚙️ 1. 算術演算子：数値を計算する

算術演算子は、主に数値の計算に使われます。PHPの算術演算子とほとんど同じです。

演算子 説明 例
+ 加算 5 + 3 → 8
- 減算 5 - 3 → 2
* 乗算 5 * 3 → 15
/ 除算 6 / 3 → 2
% 剰余（余り） 5 % 3 → 2
** べき乗 (ES2016) 5 ** 3 → 125 (5の3乗)

文字列の連結にも+演算子が使われます。

const lastName = '山田';
const firstName = '太郎';
const fullName = lastName + ' ' + firstName; // "山田 太郎"

⚙️ 2. 比較演算子：2つの値を比べる

比較演算子は、2つの値を比較し、その結果を真偽値（trueまたはfalse）で返します。条件分岐（if文）などで頻繁に使われます。

演算子 説明 例
> より大きい 5 > 3 → true
< より小さい 5 < 3 → false
>= 以上 5 >= 5 → true
<= 以下 5 <= 3 → false
== 等価（値が等しい） 5 == "5" → true ⚠️
!= 不等価（値が等しくない） 5 != "5" → false ⚠️
=== 厳密等価（値と型が等しい） 5 === "5" → false ✅
!== 厳密不等価（値または型が等しくない） 5 !== "5" → true ✅

⚠️ 最重要： == ではなく === を使おう

JavaScriptを学ぶ上で、最も重要な習慣の一つが「常に===（厳密等価演算子）を使う」ことです。

  • ==（等価演算子）は、比較する際に暗黙的な型変換を行います。例えば、数値の5と文字列の"5"を比較すると、JavaScriptが自動で文字列を数値に変換しようとするため、trueという結果になります。これは予期せぬバグの温床となります。

  • ===（厳密等価演算子）は、型変換を行いません。値とデータ型の両方が完全に一致している場合のみtrueを返します。数値の5と文字列の"5"はデータ型が異なるため、falseとなります。こちらの方が、はるかに厳密で安全です。

比較 == (等価) === (厳密等価) 理由
5と"5" true false 型が違う (number vs string)
0とfalse true false 型が違う (number vs boolean)
nullとundefined true false 型が違う (object vs undefined)

結論：特別な理由がない限り、比較には常に === と !== を使用してください。 これは、JavaScriptコミュニティにおける広く確立されたベストプラクティスです。

⚙️ 3. 論理演算子：複数の条件を組み合わせる

論理演算子は、複数の真偽値を組み合わせて、新しい真偽値を作り出します。if文の条件を複雑にする際に使われます。

演算子 名前 説明 例
&& AND (かつ) 両方の条件がtrueの場合のみtrueを返す。 (5 > 3) && (1 < 2) → true
|| OR (または) どちらか一方の条件でもtrueであればtrueを返す。 (5 < 3) || (1 < 2) → true
! NOT (ではない) 真偽値を反転させる。 !(5 > 3) → false

ショートサーキット評価

&&と||には、「ショートサーキット評価」という重要な性質があります。

  • && (AND): 左側の条件がfalseだった場合、右側の条件は評価（実行）されません。なぜなら、結果がfalseになることが確定するからです。
  • || (OR): 左側の条件がtrueだった場合、右側の条件は評価（実行）されません。なぜなら、結果がtrueになることが確定するからです。

これは、単なるパフォーマンス最適化だけでなく、エラー回避のテクニックとしても使われます。

const user = null;

// userがnullなので、user.isAdminの評価は実行されず、エラーにならない
if (user && user.isAdmin) {
  console.log('管理者です');
}

✨ まとめ

  • 算術演算子（+, -, *, /, %）で基本的な計算を行う。
  • 比較演算子（>, <, >=, <=）で大小を比較する。
  • 等価比較では、暗黙の型変換を避けるため、常に厳密等価演算子===と!==を使用する。
  • 論理演算子（&&, ||, !）で、複数の条件を組み合わせる。
