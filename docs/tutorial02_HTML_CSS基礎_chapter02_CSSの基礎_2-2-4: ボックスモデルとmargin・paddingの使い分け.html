Chapter 2: CSSの基礎

Section 3: ボックスモデルとmargin/paddingの使い分け

🖼️ 完成形のイメージ

このセクションでは、CSSレイアウトの根幹である「ボックスモデル」を学びます。要素の「内側の余白（padding）」と「外側の余白（margin）」を使い分けることで、以下のようにコンテンツと要素間のスペースを自在にコントロールする方法を理解します。

ここにスクリーンショットを挿入: 2つのボックスが並んでおり、片方のボックスはpaddingで内側のテキストとの間に余白があり、もう片方はmarginで隣のボックスとの間に距離が空いている様子がわかる図

🎯 このセクションで学ぶこと

このセクションでは、すべてのHTML要素が持つ「ボックス」としての性質と、そのスペースを管理するためのmarginとpaddingプロパティについて学びます。このセクションを終えると、あなたは以下のことができるようになります。

  • CSSのボックスモデル（content, padding, border, margin）を図解で説明できるようになる。
  • padding（内側の余白）とmargin（外側の余白）の役割の違いを明確に説明できる。
  • 要素の内部のスペースを空けたい時と、要素間の距離を取りたい時に、paddingとmarginを正しく使い分けられるようになる。

導入

Webページ上のすべてのHTML要素は、ブラウザによって長方形の「ボックス」として描画されます。CSSでレイアウトを組むということは、これらのボックスの大きさや配置をコントロールすることに他なりません。このボックスの構造を理解するための概念が「ボックスモデル」です。そして、そのボックスの余白を操る上で最も重要なのが padding と margin です。この二つの違いをマスターすることは、CSSレイアウトの第一歩です。

概念の説明

CSSのボックスモデルは、一つのボックスが以下の4つの領域で構成されていると考えます。

  1. Content（コンテンツ）: テキストや画像など、要素の実際の内容が表示される中心領域。
  2. Padding（パディング）: コンテンツとボーダー（枠線）の間の「内側の余白」。
  3. Border（ボーダー）: パディングの外側を囲む「枠線」。
  4. Margin（マージン）: ボーダーの外側にある「外側の余白」。この余白は透明で、他の要素との距離を定義します。

図解：ボックスモデル

+-------------------------------------------+
|                  Margin                   |
|  +-------------------------------------+  |
|  |                Border               |  |
|  |  +-------------------------------+  |  |
|  |  |            Padding            |  |  |
|  |  |  +-------------------------+  |  |  |
|  |  |  |         Content         |  |  |  |
|  |  |  +-------------------------+  |  |  |
|  |  |                               |  |  |
|  |  +-------------------------------+  |  |
|  |                                     |  |
|  +-------------------------------------+  |
|                                           |
+-------------------------------------------+

padding と margin の最も重要な違いは、ボーダー（枠線）の内側にあるか、外側にあるかです。

  • Padding: ボックスの内側の余白。paddingを増やすと、コンテンツは小さいまま、ボックス自体が大きくなります。背景色や背景画像はpadding領域にも適用されます。
  • Margin: ボックスの外側の余白。marginを増やしても、ボックス自体の大きさは変わりません。隣接する他の要素との距離が広がります。

詳細解説

paddingとmarginの具体的な使い方と、その挙動の違いを見ていきましょう。

padding の使い方

paddingは、コンテンツが窮屈に見えないように、要素の内部にスペースを作りたい時に使います。例えば、ボタンのテキストの周りに余白を持たせて、クリックしやすく見栄えを良くする、といった場面です。

  • コード例:
    html <style> .button { background-color: dodgerblue; color: white; border: 2px solid navy; padding: 10px 20px; /* 上下10px, 左右20pxのpadding */ } </style> <button class="button">クリックしてください</button>
  • コードリーディング:
    • padding: 10px 20px; → paddingプロパティに値を2つ指定すると、1つ目が「上下」、2つ目が「左右」の余白になります。
    • この指定により、ボタンのテキスト「クリックしてください」の上下に10px、左右に20pxの余白ができます。
    • この余白部分にも、背景色であるdodgerblueが適用されている点に注目してください。

margin の使い方

marginは、要素と要素の間に距離を取りたい、レイアウト上の間隔を調整したい時に使います。例えば、2つのボタンがくっつきすぎないように、間にスペースを空ける、といった場面です。

  • コード例:
    html <style> .box { width: 100px; height: 100px; background-color: tomato; border: 2px solid black; } .box-2 { margin-top: 30px; /* 上に30pxのmargin */ } </style> <div class="box">ボックス1</div> <div class="box box-2">ボックス2</div>
  • コードリーディング:
    • .box-2 に margin-top: 30px; を指定しています。
    • これにより、「ボックス2」の上側に30pxの透明な余白ができ、真上にある「ボックス1」との間に30pxの距離が生まれます。
    • ボックス自体の大きさ（100px × 100px）は変わっていない点に注目してください。

値の指定方法（ショートハンド）

paddingとmarginは、上下左右の値をまとめて指定する「ショートハンド」という書き方ができます。これは非常によく使われるので必ず覚えましょう。

値の数 適用順 例 (padding: 10px 20px 30px 40px;)
4つ 上 → 右 → 下 → 左（時計回り） 上:10px, 右:20px, 下:30px, 左:40px
3つ 上 → 左右 → 下 上:10px, 左右:20px, 下:30px
2つ 上下 → 左右 上下:10px, 左右:20px
1つ 上下左右すべて 上下左右すべて:10px

💡 TIP

  • marginの相殺: 隣接するブロックレベル要素の上下のmarginは、大きい方の値に「相殺」されるという特殊なルールがあります。例えば、上の要素のmargin-bottomが20px、下の要素のmargin-topが30pxの場合、間の余白は50pxではなく、30pxになります。これは最初は混乱するかもしれませんが、そういうものだと覚えておきましょう。
  • box-sizingプロパティ: 古いブラウザでは、widthにpaddingとborderが含まれないため、計算が複雑でした。現在では、CSSの冒頭で * { box-sizing: border-box; } と指定するのが一般的です。これにより、widthやheightにpaddingとborderが含まれるようになり、直感的なサイズ指定が可能になります。この設定は、後のチュートリアルで詳しく扱います。

✨ まとめ

  • すべてのHTML要素は、Content, Padding, Border, Marginから成る「ボックス」である。
  • paddingはボーダーの内側の余白で、要素の大きさに影響する。
  • marginはボーダーの外側の余白で、他の要素との距離を調整する。
  • 「内側の余白」にはpadding、「外側の距離」にはmarginと使い分ける。
  • 値の指定は、時計回りに「上→右→下→左」の順番で覚える。
