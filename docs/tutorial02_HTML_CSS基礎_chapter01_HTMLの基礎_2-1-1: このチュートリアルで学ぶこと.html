🎯 このセクションで学ぶこと

このセクションでは、Webページの「骨格」を作るHTMLと、「見た目」を装飾するCSSの基本的な役割分担について学びます。このセクションを終えると、あなたは以下のことができるようになります。

  • HTMLがWebページの「構造」を定義する言語であることを説明できるようになる。
  • CSSがWebページの「見た目」を定義する言語であることを説明できるようになる。
  • なぜ「構造と見た目の分離」が重要なのか、そのメリットを説明できるようになる。

導入：なぜHTML/CSSから学ぶのか？

前のチュートリアルでは、Webがどのように動いているかの裏側の仕組みを探検しました。このチュートリアルからは、いよいよWebページの「表側」、つまりユーザーが直接目にする部分の作成に入ります。その主役となるのが、HTMLとCSSです。

Laravelエンジニアであるあなたは、Bladeテンプレートを使ってHTMLを生成することに慣れているかもしれません。しかし、フロントエンドの世界では、HTMLとCSSの深い理解が、よりリッチでインタラクティブなUIを構築するための絶対的な土台となります。このチュートリアルでは、改めてHTMLとCSSの基礎を固め、その役割と関係性を明確に理解することを目指します。

概念の説明

HTMLとCSSの関係は、よく「人間」や「建物」に例えられます。

  • HTML (HyperText Markup Language): Webページの骨格や構造を定義する言語です。人間で言えば「骨格」、建物で言えば「鉄骨や柱」に相当します。見出し、段落、リスト、画像、リンクなど、ページに「何が」存在するかを意味づけします。

  • CSS (Cascading Style Sheets): HTMLで定義された骨格に対して、見た目や装飾を施す言語です。人間で言えば「服装、髪型、メイク」、建物で言えば「壁紙、塗装、インテリア」に相当します。文字の色、大きさ、背景、配置など、ページが「どのように」見えるかを定義します。

重要なのは、HTMLが構造を、CSSが見た目を担当するという役割分担です。この「構造と見た目の分離」は、メンテナンス性が高く、効率的なWeb開発を行うための非常に重要な原則です。

詳細解説

HTMLの役割：文書の構造化

HTMLは「マークアップ言語」と呼ばれます。これは、文章の各部分が「これは見出しです」「これは段落です」といったように、「意味（セマンティクス）」をマーク（目印）付けしていくことから来ています。

例えば、以下のような簡単なHTMLを見てみましょう。

<!DOCTYPE html>
<html>
<head>
  <title>私の最初のページ</title>
</head>
<body>
  <h1>こんにちは、世界！</h1>
  <p>これは私の最初のWebページです。</p>
</body>
</html>

このコードは、ブラウザに対して以下のような情報を伝えています。

  • <h1>: 「こんにちは、世界！」というテキストは、このページで最も重要なレベル1の見出しです。
  • <p>: 「これは私の最初のWebページです。」というテキストは、一つの段落です。

この時点では、文字の色や大きさといった「見た目」の情報は一切含まれていません。ただ、文書の構造だけが定義されています。

CSSの役割：見た目の装飾

次に、CSSを使ってこのHTMLにスタイルを適用してみましょう。CSSは、どの要素に（セレクタ）、どのようなスタイルを（プロパティと値）適用するかを記述します。

h1 {
  color: blue;
  font-size: 32px;
}

p {
  color: gray;
}

このCSSは、ブラウザに対して以下のように指示します。

  • h1: <h1>タグで囲まれた要素の文字色を青（blue）に、文字サイズを32ピクセル（32px）にしなさい。
  • p: <p>タグで囲まれた要素の文字色を灰色（gray）にしなさい。

HTMLとCSSの連携

HTMLファイルからこのCSSファイルを読み込むことで、初めて構造と見た目が結びつきます。

<!DOCTYPE html>
<html>
<head>
  <title>私の最初のページ</title>
  <link rel="stylesheet" href="style.css"> <!-- この行でCSSファイルを読み込む -->
</head>
<body>
  <h1>こんにちは、世界！</h1>
  <p>これは私の最初のWebページです。</p>
</body>
</html>

こうすることで、ブラウザはまずHTMLを解釈してページの骨格を作り、次にCSSを解釈してその骨格にデザインを施し、最終的にスタイルが適用されたWebページを画面に表示するのです。

💡 TIP

  • なぜ分離するのか？: もしHTMLに直接スタイルを書き込んでしまうと（古い手法では可能でした）、サイト全体のデザインを変更したい場合に、すべてのHTMLファイルを一つ一つ修正する必要があり、非常に手間がかかります。CSSファイルを一つ修正するだけでサイト全体の見た目を変更できる「構造と見た目の分離」は、大規模な開発において不可欠です。
  • セマンティックHTML: HTMLタグは、その意味に合ったものを使うことが非常に重要です。これを「セマンティックHTML」と呼びます。例えば、単に文字を太くしたいだけで見出しタグ<h1>を使うべきではありません。これは、検索エンジンがページの構造を理解する上でも重要になります（SEO）。

✨ まとめ

  • HTMLはWebページの「骨格」や「構造」を定義する。
  • CSSはHTMLで作られた骨格に「見た目」や「装飾」を施す。
  • 「構造と見た目の分離」は、効率的でメンテナンス性の高いWeb開発の基本原則である。
  • ブラウザはHTMLを解釈して構造を作り、CSSを解釈してスタイルを適用し、最終的なページを表示する。
