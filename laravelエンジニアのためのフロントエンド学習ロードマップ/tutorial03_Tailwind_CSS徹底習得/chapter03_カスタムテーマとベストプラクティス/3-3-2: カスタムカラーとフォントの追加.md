# 3-3-2: カスタムカラーとフォントの追加

## 🎯 このセクションで学ぶこと

- `tailwind.config.js` を編集して、独自のカラーパレットを追加できるようになる
- プロジェクトで使用するカスタムフォントを定義できるようになる
- `theme.extend` の役割を理解し、既存のテーマを拡張できるようになる

## 🖼️ 完成イメージ

このセクションでは、`tailwind.config.js` をカスタマイズして、プロジェクト独自のブランドカラーやフォントを定義します。

## 導入

Tailwind CSSは豊富なデフォルトカラーやフォントを提供していますが、実際のプロジェクトではブランドガイドラインに沿った独自のデザイントークンが必要になります。`tailwind.config.js` を編集することで、Tailwindの機能を拡張し、プロジェクトに最適化できます。

## 詳細解説

### `tailwind.config.js` の `theme` オブジェクト

カスタマイズのほとんどは `theme` オブジェクト内で行います。`theme` オブジェクトに直接キーを追加すると、デフォルトのテーマが上書きされます。既存のテーマを維持しつつ追加したい場合は、`theme.extend` を使います。

### カスタムカラーの追加

`theme.extend.colors` に独自のカラーを追加します。キーがクラス名の一部になり、値がカラーコードになります。

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        'custom-blue': '#1E40AF',
        'custom-green': {
          light: '#6EE7B7',
          DEFAULT: '#10B981',
          dark: '#047857',
        },
      },
    },
  },
}
```

上記のように設定すると、`bg-custom-blue` や `text-custom-green-light` のようなクラスが使えるようになります。

### カスタムフォントの追加

`theme.extend.fontFamily` にフォントファミリーを追加します。キーがクラス名の一部になり、値がフォントファミリーの配列になります。

```javascript
// tailwind.config.js
const defaultTheme = require('tailwindcss/defaultTheme')

module.exports = {
  theme: {
    extend: {
      fontFamily: {
        sans: ['Inter var', ...defaultTheme.fontFamily.sans],
        mono: ['Fira Code', ...defaultTheme.fontFamily.mono],
      },
    },
  },
}
```

`defaultTheme` をインポートしてスプレッド構文で展開することで、デフォルトのフォントファミリーを維持しつつ、先頭にカスタムフォントを追加できます。

## 💡 TIP

- `tailwind.config.js` を変更した後は、開発サーバーを再起動しないと変更が反映されない場合があります。

## 🏃 実践

前のセクションで作成したプロジェクトに、独自のブランドカラーとフォントを追加してみましょう。

## ✨ まとめ

- `tailwind.config.js` の `theme.extend` を使ってテーマを拡張する
- `colors` オブジェクトに独自のカラーパレットを追加できる
- `fontFamily` オブジェクトにカスタムフォントを追加できる
