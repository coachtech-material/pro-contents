# Tutorial 3: Tailwind CSS徹底習得

## Chapter 3: Tailwind CSSのカスタマイズ

### Chapter 3 ハンズオン: プロジェクトにブランドカラーを導入し、ボタンコンポーネントを作成する

🖼️ **完成形のイメージ**

このハンズオンでは、`tailwind.config.js`をカスタマイズしてプロジェクト独自のブランドカラーを定義し、さらに`@apply`を使って、そのブランドカラーを適用した再利用可能なボタンコンポーネントを作成します。これにより、Tailwindをプロジェクトのデザインシステムとして活用する第一歩を踏み出します。

*ここにスクリーンショットを挿入: `tailwind.config.js`に`pro-contents-blue`のようなカスタムカラーが定義されているコード。`input.css`で`.btn-pro`クラスに`@apply bg-pro-contents-blue ...`が適用されているコード。そしてHTML上で、`<button class="btn-pro">`と書かれたボタンが、定義したブランドカラーで表示されている様子。*

--- 

🎯 **このセクションで学ぶこと**

このハンズオンを終えると、あなたは以下のことができるようになります。

-   `tailwind.config.js`の`theme.extend.colors`を編集して、プロジェクト固有のカラーパレットを追加できるようになる。
-   `@apply`を使って、カスタムカラーを適用したコンポーネントクラス（`.btn-brand`など）を作成できるようになる。
-   HTMLのクラス名を、汎用的なユーティリティクラスから、より意味的なコンポーネントクラスへとリファクタリングできるようになる。

--- 

### 🏃 実践: 一緒に作ってみましょう！

Chapter 2で作成した製品カードのプロジェクトを引き続き使用します。現在の「カートに追加」ボタンは、`bg-blue-500`という汎用的な青色を使っていますが、これをプロジェクトのブランドカラーに変更し、さらにコンポーネント化することで、将来のメンテナンス性を向上させます。

#### 💭 実装の思考プロセス

**このハンズオンのゴール:**

1.  **ブランドカラーの定義:** `tailwind.config.js`を編集し、「Pro-Contents」のブランドカラーとして、少し落ち着いた青色を`pro-blue`という名前で登録する。
2.  **ボタンコンポーネントの作成:** `src/input.css`に、`.btn`というベースクラスと、`.btn-pro`という修飾クラスを`@apply`を使って定義する。`.btn-pro`には、先ほど定義した`bg-pro-blue`を適用する。
3.  **HTMLのリファクタリング:** 製品カード内のボタンの長いクラスリストを、新しく作成した`.btn`と`.btn-pro`クラスに置き換える。

💡 **ポイント:** デザインシステムを構築する上で、色に意味的な名前（`primary`, `brand-color`など）を付けることは非常に重要です。これにより、将来「ブランドカラーを変更したい」となった時に、`tailwind.config.js`の1箇所を修正するだけで、サイト全体の関連するすべての要素の色を一括で変更できるようになります。

#### 📝 ステップバイステップで実装

##### Step 1: ブランドカラーを`tailwind.config.js`に追加する

まず、プロジェクトのブランドカラーを定義します。ここでは、`#2563EB`という青色を`pro-blue`として登録しましょう。

1-1. `tailwind.config.js`ファイルを開き、`theme.extend.colors`オブジェクトを編集します。

-   **コマンド/コード (tailwind.config.js):**
    ```javascript
    /** @type {import("tailwindcss").Config} */
    module.exports = {
      content: ["./*.html"],
      theme: {
        extend: {
          colors: {
            'pro-blue': '#2563EB',
          },
        },
      },
      plugins: [],
    }
    ```

1-2. 設定ファイルを変更したので、`npm run watch`で実行中のビルドプロセスを一度`Ctrl + C`で停止し、再度`npm run watch`を実行してビルドを再開します。これを忘れると、新しい色が反映されません。

##### Step 2: `@apply`でボタンコンポーネントを作成する

次に、`src/input.css`に、再利用可能なボタンクラスを作成します。すべてのボタンに共通するスタイルを`.btn`に、ブランドカラーを適用するスタイルを`.btn-pro`に分離して定義するのが良いプラクティスです。

2-1. `src/input.css`ファイルを開き、末尾に以下のコードを追加します。

-   **コマンド/コード (src/input.css):**
    ```css
    /* ... 既存の@tailwindディレクティブ ... */

    @layer components {
      .btn {
        @apply font-bold py-2 px-4 rounded-full transition-colors duration-300;
      }

      .btn-pro {
        @apply bg-pro-blue text-white;
      }

      .btn-pro:hover {
        @apply bg-blue-700; /* ホバー時は少し濃い色に */
      }
    }
    ```

-   **コードリーディング:**
    -   `.btn`クラスには、フォントの太さ、余白、角丸、トランジションといった、ボタンの形状や挙動に関する共通スタイルをまとめています。
    -   `.btn-pro`クラスには、配色に関するスタイル（背景色と文字色）をまとめています。ここで、先ほど定義した`bg-pro-blue`を早速使っています。
    -   このようにスタイルを分離することで、例えば「形状は同じで色が違うボタン（`.btn-secondary`など）」を簡単に追加できるようになります。

##### Step 3: HTMLをリファクタリングする

最後に、製品カードのHTMLを編集し、新しいコンポーネントクラスを使うように書き換えます。

3-1. `index.html`を開き、製品カード内の`<button>`要素を見つけます。

-   **コマンド/コード (index.html):**
    ```html
    <!-- 変更前 -->
    <button class="bg-blue-500 text-white py-2 px-4 rounded-full hover:bg-blue-600 transition-colors">カートに追加</button>

    <!-- 変更後 -->
    <button class="btn btn-pro">カートに追加</button>
    ```

3-2. すべての製品カードのボタンを同様に修正します。

#### ✨ 完成！

お疲れ様でした！ブラウザをリロードして確認してください。ボタンの青色が、定義したブランドカラー（`#2563EB`）に変わっているはずです。そして、HTMLのクラス名が非常にシンプルで意味的になったことにも注目してください。

あなたは今、Tailwind CSSを単なるユーティリティ集として使うだけでなく、プロジェクトのデザインシステムを構築するためのツールとして活用する、より高度なスキルを身につけました。この考え方は、大規模なプロジェクトになるほど、その真価を発揮します。
