# 6-1-3: 基本的な型（string, number, boolean, array, object）

## 🎯 この章で学ぶこと

- TypeScriptの基本的な型を理解する
- 配列とオブジェクトの型定義方法を学ぶ
- `interface`と`type`の使い分けを理解する

## はじめに

TypeScriptには、JavaScriptのプリミティブ型に対応する基本的な型が用意されています。このセクションでは、最もよく使う型について学んでいきます。

## 1. プリミティブ型

### string型

文字列を表す型です。シングルクォート、ダブルクォート、バッククォートのいずれでも定義できます。

```typescript
const name: string = "Taro";
const greeting: string = `Hello, ${name}!`;
```

### number型

数値を表す型です。整数も小数も同じ`number`型で扱います。

```typescript
const age: number = 25;
const price: number = 1980.5;
const hex: number = 0xff; // 16進数
```

### boolean型

真偽値（`true`または`false`）を表す型です。

```typescript
const isActive: boolean = true;
const hasPermission: boolean = false;
```

## 2. 配列の型

配列の型を定義するには、2つの書き方があります。

```typescript
// 書き方1: 型名[]
const numbers: number[] = [1, 2, 3, 4, 5];
const names: string[] = ["Taro", "Hanako", "Jiro"];

// 書き方2: Array<型名>（ジェネリクス構文）
const scores: Array<number> = [80, 90, 75];
```

どちらの書き方も同じ意味ですが、`型名[]`の方がよく使われます。

## 3. オブジェクトの型

オブジェクトの型を定義するには、`interface`または`type`を使います。

### interfaceを使う方法

```typescript
interface User {
  name: string;
  age: number;
  email: string;
}

const user: User = {
  name: "Taro",
  age: 25,
  email: "taro@example.com"
};
```

### typeを使う方法

```typescript
type Product = {
  id: number;
  name: string;
  price: number;
};

const product: Product = {
  id: 1,
  name: "TypeScript入門",
  price: 2980
};
```

### interfaceとtypeの違い

`interface`と`type`は似ていますが、いくつかの違いがあります。

| 特徴 | interface | type |
|------|-----------|------|
| 拡張方法 | `extends`キーワード | `&`（Intersection） |
| 同名の宣言 | マージされる | エラーになる |
| プリミティブ型のエイリアス | 不可 | 可能 |
| Union型の定義 | 不可 | 可能 |

一般的には、オブジェクトの形状を定義する場合は`interface`を、それ以外の型定義には`type`を使うことが多いです。

## 4. オプショナルプロパティ

プロパティ名の後ろに`?`を付けると、そのプロパティは省略可能になります。

```typescript
interface User {
  name: string;
  age?: number; // 省略可能
  email?: string; // 省略可能
}

const user1: User = { name: "Taro" }; // OK
const user2: User = { name: "Hanako", age: 30 }; // OK
const user3: User = { name: "Jiro", age: 28, email: "jiro@example.com" }; // OK
```

## 5. 読み取り専用プロパティ

`readonly`キーワードを付けると、そのプロパティは変更できなくなります。

```typescript
interface Config {
  readonly apiUrl: string;
  readonly timeout: number;
}

const config: Config = {
  apiUrl: "https://api.example.com",
  timeout: 5000
};

// config.apiUrl = "https://other.com"; // エラー！読み取り専用
```

## 6. any型とunknown型

型が不明な場合に使う型として、`any`と`unknown`があります。

```typescript
// any型: 何でも代入でき、何にでも代入できる（型チェックを無効化）
let anyValue: any = "hello";
anyValue = 123; // OK
anyValue = true; // OK

// unknown型: 何でも代入できるが、使用時に型チェックが必要
let unknownValue: unknown = "hello";
// console.log(unknownValue.length); // エラー！型が不明
if (typeof unknownValue === "string") {
  console.log(unknownValue.length); // OK（型が絞り込まれた）
}
```

`any`型は型チェックを完全に無効化するため、使用は避けるべきです。型が不明な場合は`unknown`を使い、適切に型を絞り込むことが推奨されます。

## ✨ まとめ

TypeScriptの基本的な型として、`string`、`number`、`boolean`、配列、オブジェクトの型定義方法を学びました。オブジェクトの型は`interface`または`type`で定義でき、オプショナルプロパティや読み取り専用プロパティも指定できます。`any`型は型チェックを無効化するため、代わりに`unknown`型を使うことが推奨されます。
