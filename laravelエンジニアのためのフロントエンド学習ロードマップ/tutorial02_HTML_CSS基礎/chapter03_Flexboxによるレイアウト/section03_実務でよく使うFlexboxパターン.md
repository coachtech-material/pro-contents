# Tutorial 2: HTML/CSS基礎

## Chapter 3: Flexboxによるレイアウト

### Section 3: 実務でよく使うFlexboxパターン

🖼️ **完成形のイメージ**

このセクションでは、これまでに学んだFlexboxの知識を応用して、実際のWebサイトで頻繁に登場する実践的なレイアウトパターンを構築します。ヘッダー、カードリスト、フォームなど、様々なコンポーネントがFlexboxでいかに効率的に作れるかを学びます。

*ここにスクリーンショットを挿入: (1)左にロゴ、右にナビゲーションメニューが配置されたヘッダー (2)レスポンシブ対応のカードが並んだリスト (3)ラベルと入力欄が綺麗に揃ったフォーム*

--- 

🎯 **このセクションで学ぶこと**

このセクションでは、Flexboxのプロパティを組み合わせて、実務で頻出するUIコンポーネントを構築するテクニックを学びます。このセクションを終えると、あなたは以下のことができるようになります。

- `justify-content: space-between;` を使って、典型的なヘッダーレイアウトを作成できるようになる。
- `flex-wrap: wrap;` を使って、画面幅に応じて折り返すレスポンシブなカードリストを作成できるようになる。
- `flex-grow` や `flex-basis` を使って、アイテムの幅を柔軟に制御できるようになる。

--- 

### 導入

Flexboxの個々のプロパティを学んだだけでは、まだ点と点です。このセクションでは、それらの知識を線で結び、実際のUIコンポーネントという「面」を構築していきます。「ヘッダーの右端にメニューを配置したい」「カードを横に並べたいけど、画面が狭くなったら折り返してほしい」といった、実務で必ず遭遇する要求に、Flexboxを使ってどう応えるかをパターンとして学びましょう。

### 詳細解説

#### パターン1: ヘッダーレイアウト（ロゴとナビゲーション）

Webサイトの最上部に表示されるヘッダーで、左側にロゴ、右側にナビゲーションメニューを配置する、という最も一般的なレイアウトです。

- **要件:** ロゴは左端、ナビゲーションメニューは右端に配置したい。
- **使うプロパティ:** `display: flex;`, `justify-content: space-between;`, `align-items: center;`

- **HTML:**
  ```html
  <header class="site-header">
    <div class="logo">My Site</div>
    <nav>
      <ul>
        <li><a href="#">ホーム</a></li>
        <li><a href="#">サービス</a></li>
        <li><a href="#">お問い合わせ</a></li>
      </ul>
    </nav>
  </header>
  ```
- **CSS:**
  ```css
  .site-header {
    display: flex;
    justify-content: space-between; /* 両端に配置 */
    align-items: center; /* 垂直方向中央揃え */
    padding: 10px 20px;
    background-color: #333;
    color: white;
  }
  .site-header nav ul {
    display: flex; /* ナビゲーションの項目も横並びに */
    list-style: none;
    padding: 0;
    gap: 15px; /* 項目間の隙間 */
  }
  ```
- **ポイント:** 親要素である`.site-header`に`justify-content: space-between;`を指定するだけで、2つの子要素（ロゴとナビゲーション）が左右の両端に綺麗に分かれます。また、ナビゲーションメニュー自体もFlexboxを使って項目を横並びにしています（Flexboxの入れ子）。

#### パターン2: レスポンシブなカードリスト

商品一覧やブログ記事一覧などで使われる、複数のカードを並べるレイアウトです。画面幅が広い時は横に並び、狭くなると自動的に折り返して縦に並びます。

- **要件:** カードを横に並べ、コンテナの幅を超えたら自動で次の行に折り返したい。
- **使うプロパティ:** `display: flex;`, `flex-wrap: wrap;`, `gap`

- **HTML:**
  ```html
  <div class="card-container">
    <div class="card">カード 1</div>
    <div class="card">カード 2</div>
    <div class="card">カード 3</div>
    <div class="card">カード 4</div>
  </div>
  ```
- **CSS:**
  ```css
  .card-container {
    display: flex;
    flex-wrap: wrap; /* 折り返しを許可 */
    gap: 20px; /* カード間の隙間 */
  }
  .card {
    flex-basis: 300px; /* カードの基本幅を指定 */
    /* ...カードの装飾スタイル... */
  }
  ```
- **ポイント:** `flex-wrap: wrap;`が鍵です。これを指定しないと、アイテムは無理やり一行に収まろうとして縮んでしまいます。`wrap`を指定することで、コンテナの幅が足りなくなるとアイテムが自然に下の行に落ちてくれます。`gap`でアイテム間の隙間を簡単に指定できるのも便利です。

#### パターン3: アイテム幅の柔軟な制御

サイドバーとメインコンテンツのように、一部の幅は固定し、残りの部分が利用可能なスペースをすべて埋める、といったレイアウトです。

- **要件:** 片方のアイテム（サイドバー）の幅は固定し、もう片方（メインコンテンツ）が残りの幅をすべて使うようにしたい。
- **使うプロパティ:** `display: flex;`, `flex-grow: 1;`

- **HTML:**
  ```html
  <div class="container">
    <main class="main-content">メインコンテンツ</main>
    <aside class="sidebar">サイドバー</aside>
  </div>
  ```
- **CSS:**
  ```css
  .container {
    display: flex;
  }
  .main-content {
    flex-grow: 1; /* 残りのスペースをすべて埋める */
    background-color: lightblue;
  }
  .sidebar {
    flex-basis: 200px; /* 幅を200pxに固定 */
    background-color: lightgreen;
  }
  ```
- **ポイント:** `flex-grow`は、コンテナ内に余ったスペースがある場合に、そのアイテムがどれだけの割合でスペースを「成長（grow）」して埋めるかを指定するプロパティです。`1`を指定された`.main-content`が、余ったスペースを独り占めして伸びるため、このようなレイアウトが実現できます。

### 💡 TIP

- **アイテムの順序変更:** `order`プロパティを使うと、HTMLの構造を変えずに、Flexアイテムの表示順序をCSSだけで変更できます。例えば、スマートフォン表示の時だけ特定の要素を一番上に表示する、といったことが可能です。`order: -1;`と指定すると、他のアイテムより前に表示されます。
- **Flexboxデバッグツール:** ChromeやFirefoxの開発者ツールには、Flexboxのレイアウトを視覚的に確認・デバッグするための非常に強力な機能が備わっています。コンテナやアイテムのサイズ、余白、揃え方などが一目でわかるので、積極的に活用しましょう。

### ✨ まとめ

- ヘッダーレイアウトは `justify-content: space-between;` の鉄板パターン。
- 自動で折り返すリストは `flex-wrap: wrap;` で実現できる。
- `flex-grow: 1;` を使うと、特定のアイテムに残りのスペースをすべて埋めさせることができる。
- Flexboxは入れ子にすることで、複雑なレイアウトもコンポーネント単位でシンプルに構築できる。
